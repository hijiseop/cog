# 창고 물품 관리 대장 — 제작 가이드

태블릿으로 입고 · 출고 · 대여를 버튼 한 번에 기록하고, 재고가 자동 계산되며, 거래가 생길 때마다 알림 메일이 발송되는 시스템을 **직접 만드는 단계별 가이드**입니다.

---

## 1. 어떤 방식으로 만드나요?

요청하신 조건은 다음과 같습니다.

- 태블릿에서 사용 (앱처럼 동작)
- 입고 / 출고 / 대여 버튼 → 이름·수량 입력 → 스프레드시트에 자동 기록
- 재고 자동 계산
- 엑셀(스프레드시트)에 저장
- 거래 발생 시 간단한 알림 메일 발송

이 조건을 모두 만족하면서 **추가 비용 없이 개인이 직접 만들 수 있는 방법**은 **Google 스프레드시트 + Apps Script(웹앱)** 조합입니다.

### 왜 일반 엑셀(.xlsx)이 아닌 Google 스프레드시트인가요?

| 구분 | Google 스프레드시트 + Apps Script | 데스크톱 엑셀 + VBA 매크로 |
|---|---|---|
| 태블릿 사용 | 브라우저로 바로 사용, 홈 화면에 앱처럼 추가 가능 | 태블릿용 엑셀은 매크로(VBA) 실행 불가 |
| 버튼식 입력 화면 | HTML 화면을 자유롭게 제작 가능 | 모바일에서 매크로 버튼 동작 안 함 |
| 메일 자동 발송 | 내장 기능으로 무료 발송 | Power Automate 등 별도 유료/복잡 설정 필요 |
| 비용 | 무료 | 무료지만 자동화 환경 구축이 어려움 |
| 엑셀 호환 | 언제든 `.xlsx`로 다운로드 가능 | — |

**결론:** Google 스프레드시트는 화면상 엑셀과 똑같이 동작하고, 필요하면 `파일 → 다운로드 → Microsoft Excel(.xlsx)`로 엑셀 파일을 그대로 받을 수 있습니다. 즉 "엑셀"의 장점은 그대로 쓰면서 태블릿·앱·메일 자동화를 모두 해결할 수 있어, 이 가이드는 이 방식을 기준으로 작성했습니다.

> 만약 사내 규정상 반드시 데스크톱 엑셀 파일로만 관리해야 한다면, 이 시스템에서 주기적으로 `.xlsx`를 내려받아 보관하는 방식으로 병행할 수 있습니다.

## 2. 준비물

- Google 계정 1개 (없으면 [accounts.google.com](https://accounts.google.com)에서 무료 생성)
- 태블릿 (크롬 등 웹 브라우저 설치)
- PC (초기 제작은 PC에서 하는 것이 편합니다. 태블릿만 있어도 가능)
- 알림 메일을 받을 주소: `iowli@naver.com` (Naver 등 어떤 메일이든 가능)

> 시스템을 만드는 Google 계정과, 알림을 **받는** 메일 주소는 달라도 됩니다. 알림은 `iowli@naver.com`으로 받도록 설정합니다.

## 3. 전체 구조 한눈에 보기

```
[태블릿 입력 화면]            [Google 스프레드시트]
 ┌──────────────┐            ┌────────────────────┐
 │ 입고 출고 대여 반납 │  기록   │ 거래내역 시트       │
 │ 물품명 / 수량 입력 │ ───────▶ │ (모든 거래 누적)    │
 │   [기록하기]      │          ├────────────────────┤
 └──────────────┘            │ 재고현황 시트       │
        │                    │ (자동 계산)         │
        │                    └────────────────────┘
        │                              │
        └───── 거래 발생 ──────────────▶ 알림 메일 발송
                                  (iowli@naver.com)
```

- **거래내역 시트**: 입고·출고·대여·반납이 한 줄씩 차곡차곡 쌓이는 원장(原帳)
- **재고현황 시트**: 거래내역을 바탕으로 물품별 현재고가 자동 계산되는 표
- **입력 화면(웹앱)**: 태블릿에서 버튼으로 조작하는 화면
- **알림 메일**: 거래가 기록될 때마다 자동 발송

재고 계산 공식: **현재고 = 총입고 − 총출고 − 대여중(미반납 수량)**

## 4. 1단계 — 스프레드시트 만들기

### 4-1. 새 스프레드시트 생성

1. [sheets.google.com](https://sheets.google.com) 접속 → **빈 스프레드시트** 클릭
2. 좌측 상단 제목을 `창고 물품 관리 대장`으로 변경
3. 상단 메뉴 **파일 → 설정** → **시간대**를 `(GMT+09:00) 서울`로 지정 → 저장
   - 날짜·시간이 한국 시각으로 기록되게 하는 중요한 설정입니다.

### 4-2. "거래내역" 시트 만들기

화면 하단 시트 탭 이름(`시트1`)을 더블클릭해 **거래내역**으로 변경합니다. 1행에 아래 제목을 입력합니다.

| A | B | C | D | E | F | G | H | I | J |
|---|---|---|---|---|---|---|---|---|---|
| 일시 | 구분 | 물품명 | 수량 | 담당자 | 대여자 | 반납예정일 | 반납여부 | 반납일 | 메모 |

1행을 선택해 **굵게(Bold)** 처리하고, 보기 좋게 배경색을 넣어도 좋습니다.

### 4-3. "재고현황" 시트 만들기

하단의 **+** 버튼으로 시트를 추가하고 이름을 **재고현황**으로 지정합니다. 1행에 아래 제목을 입력합니다.

| A | B | C | D | E |
|---|---|---|---|---|
| 물품명 | 총입고 | 총출고 | 대여중 | 현재고 |

이 시트의 내용은 **사람이 입력하지 않습니다.** 거래가 기록될 때마다 프로그램이 자동으로 다시 계산해 채워 줍니다.

> ⚠️ 시트 이름은 반드시 `거래내역`, `재고현황` 으로 정확히 입력하세요. 띄어쓰기나 오타가 있으면 코드가 시트를 찾지 못합니다.

## 5. 2단계 — Apps Script 코드 넣기

Apps Script는 Google 스프레드시트에 내장된 자동화 기능입니다. 별도 프로그램 설치가 필요 없습니다.

1. 스프레드시트 상단 메뉴 **확장 프로그램 → Apps Script** 클릭
2. 새 탭에 코드 편집기가 열립니다. 기본으로 있는 `Code.gs`(또는 `코드.gs`) 파일의 내용을 **전부 지우고**, 아래 코드를 그대로 붙여넣습니다.

### 5-1. `Code.gs` — 기능 코드

```javascript
/*** 창고 물품 관리 대장 - 기능 코드 ***/

// ===== 설정 (여기만 필요에 맞게 수정) =====
const SHEET_TX    = '거래내역';        // 거래내역 시트 이름
const SHEET_INV   = '재고현황';        // 재고현황 시트 이름
const ADMIN_EMAIL = 'iowli@naver.com'; // 알림 메일 받을 주소
const APP_TITLE   = '창고 물품 관리 대장';

// 웹앱 첫 화면을 보여줌
function doGet() {
  return HtmlService.createHtmlOutputFromFile('index')
    .setTitle(APP_TITLE)
    .addMetaTag('viewport', 'width=device-width, initial-scale=1.0');
}

// 거래 기록 (입고 / 출고 / 대여)
function recordTransaction(data) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const tx = ss.getSheetByName(SHEET_TX);
  const now = new Date();
  const qty = Number(data.qty);

  // 입력값 검사
  if (!data.item || !qty || qty <= 0) {
    throw new Error('물품명과 수량(1 이상)을 올바르게 입력하세요.');
  }

  // 출고 / 대여 시 재고가 부족하면 막기
  if (data.type === '출고' || data.type === '대여') {
    const stock = getStock(data.item);
    if (qty > stock) {
      throw new Error('"' + data.item + '" 현재고(' + stock +
                       '개)보다 많은 수량은 처리할 수 없습니다.');
    }
  }

  // 거래내역 시트에 한 줄 추가
  tx.appendRow([
    now,                                   // A 일시
    data.type,                             // B 구분
    data.item,                             // C 물품명
    qty,                                   // D 수량
    data.handler  || '',                   // E 담당자
    data.borrower || '',                   // F 대여자
    data.dueDate  || '',                   // G 반납예정일
    data.type === '대여' ? '미반납' : '-', // H 반납여부
    '',                                    // I 반납일
    data.memo || ''                        // J 메모
  ]);

  updateInventory();  // 재고현황 다시 계산
  const stock = getStock(data.item);
  sendNotice(data.type, data.item, qty, data.handler, stock);  // 알림 메일

  return { item: data.item, stock: stock };
}

// 특정 물품의 현재고 가져오기
function getStock(item) {
  const inv = SpreadsheetApp.getActiveSpreadsheet().getSheetByName(SHEET_INV);
  const data = inv.getDataRange().getValues();
  for (let i = 1; i < data.length; i++) {
    if (data[i][0] === item) return Number(data[i][4]) || 0;
  }
  return 0;  // 거래내역이 아직 없는 새 물품
}

// 재고현황 시트를 거래내역 기준으로 다시 계산
function updateInventory() {
  const ss  = SpreadsheetApp.getActiveSpreadsheet();
  const tx  = ss.getSheetByName(SHEET_TX);
  const inv = ss.getSheetByName(SHEET_INV);
  const rows = tx.getDataRange().getValues();

  const map = {};  // 물품명 -> {입고, 출고, 대여중}
  for (let i = 1; i < rows.length; i++) {
    const type = rows[i][1];          // 구분
    const item = rows[i][2];          // 물품명
    const qty  = Number(rows[i][3]) || 0;
    const ret  = rows[i][7];          // 반납여부
    if (!item) continue;
    if (!map[item]) map[item] = { in: 0, out: 0, loan: 0 };

    if (type === '입고')      map[item].in  += qty;
    else if (type === '출고') map[item].out += qty;
    else if (type === '대여' && ret === '미반납') map[item].loan += qty;
  }

  // 표 형태로 정리
  const out = [['물품명', '총입고', '총출고', '대여중', '현재고']];
  Object.keys(map).sort().forEach(function (item) {
    const m = map[item];
    out.push([item, m.in, m.out, m.loan, m.in - m.out - m.loan]);
  });

  // 재고현황 시트 비우고 다시 쓰기
  inv.clearContents();
  inv.getRange(1, 1, out.length, 5).setValues(out);
  inv.getRange(1, 1, 1, 5).setFontWeight('bold');
}

// 미반납 대여 목록 가져오기
function getOpenLoans() {
  const tx = SpreadsheetApp.getActiveSpreadsheet().getSheetByName(SHEET_TX);
  const rows = tx.getDataRange().getValues();
  const list = [];
  for (let i = 1; i < rows.length; i++) {
    if (rows[i][1] === '대여' && rows[i][7] === '미반납') {
      let due = rows[i][6];
      if (due instanceof Date) {
        due = Utilities.formatDate(due, 'GMT+9', 'yyyy-MM-dd');
      }
      list.push({
        row: i + 1,                 // 실제 시트 행 번호
        item: rows[i][2],
        qty: rows[i][3],
        borrower: rows[i][5],
        dueDate: due || ''
      });
    }
  }
  return list;
}

// 대여품 반납 처리
function returnLoan(rowNum) {
  const tx = SpreadsheetApp.getActiveSpreadsheet().getSheetByName(SHEET_TX);
  tx.getRange(rowNum, 8).setValue('반납완료');  // H 반납여부
  tx.getRange(rowNum, 9).setValue(new Date());  // I 반납일
  const item = tx.getRange(rowNum, 3).getValue();
  const qty  = tx.getRange(rowNum, 4).getValue();

  updateInventory();
  const stock = getStock(item);
  sendNotice('반납', item, qty, '', stock);
  return { item: item, stock: stock };
}

// 알림 메일 발송 (간단 알림)
function sendNotice(type, item, qty, handler, stock) {
  try {
    const subject = '[창고 알림] ' + type + ' - ' + item + ' ' + qty + '개';
    const body =
      '창고 물품 관리 대장 알림\n' +
      '------------------------------\n' +
      '· 구분    : ' + type + '\n' +
      '· 물품명  : ' + item + '\n' +
      '· 수량    : ' + qty + '개\n' +
      (handler ? '· 담당자  : ' + handler + '\n' : '') +
      '· 처리시각: ' +
        Utilities.formatDate(new Date(), 'GMT+9', 'yyyy-MM-dd HH:mm') + '\n' +
      '· 현재고  : ' + stock + '개\n' +
      '------------------------------';
    MailApp.sendEmail(ADMIN_EMAIL, subject, body);
  } catch (e) {
    // 메일 발송이 실패해도 거래 기록 자체는 막지 않음
    console.error('메일 발송 실패: ' + e);
  }
}
```

코드 상단의 **설정 부분**만 기억하세요. 알림 받을 메일을 바꾸려면 `ADMIN_EMAIL` 값만 수정하면 됩니다.

```html
      document.getElementById('returnCard').classList.toggle('hide', !isReturn);
      document.getElementById('loanFields').classList.toggle('hide', m !== '대여');
      document.getElementById('msg').innerHTML = '';
      document.getElementById('submitBtn').innerText = m + ' 기록하기';

      if (isReturn) loadLoans();
    }

    function submitTx() {
      var data = {
        type:     mode,
        item:     document.getElementById('item').value.trim(),
        qty:      document.getElementById('qty').value,
        handler:  document.getElementById('handler').value.trim(),
        borrower: document.getElementById('borrower').value.trim(),
        dueDate:  document.getElementById('dueDate').value,
        memo:     document.getElementById('memo').value.trim()
      };
      var btn = document.getElementById('submitBtn');
      btn.disabled = true;
      btn.innerText = '저장 중...';

      google.script.run
        .withSuccessHandler(function (res) {
          showMsg('ok', '✅ ' + res.item + ' ' + mode +
                        ' 완료!  현재고: ' + res.stock + '개');
          clearForm();
          btn.disabled = false;
          btn.innerText = mode + ' 기록하기';
        })
        .withFailureHandler(function (err) {
          showMsg('err', '⚠️ ' + err.message);
          btn.disabled = false;
          btn.innerText = mode + ' 기록하기';
        })
        .recordTransaction(data);
    }

    function loadLoans() {
      document.getElementById('loanList').innerText = '불러오는 중...';
      google.script.run.withSuccessHandler(function (list) {
        if (!list.length) {
          document.getElementById('loanList').innerText =
            '미반납 대여가 없습니다.';
          return;
        }
        var html = '';
        for (var i = 0; i < list.length; i++) {
          var l = list[i];
          html += '<div class="loan-item">' +
                    '<b>' + l.item + '</b> ' + l.qty + '개<br>' +
                    '대여자: ' + (l.borrower || '-') + '<br>' +
                    '반납예정: ' + (l.dueDate || '-') + '<br>' +
                    '<button onclick="doReturn(' + l.row + ')">' +
                      '반납 처리</button>' +
                  '</div>';
        }
        document.getElementById('loanList').innerHTML = html;
      }).getOpenLoans();
    }

    function doReturn(row) {
      google.script.run
        .withSuccessHandler(function (res) {
          alert('✅ ' + res.item + ' 반납 완료!  현재고: ' +
                res.stock + '개');
          loadLoans();
        })
        .withFailureHandler(function (err) {
          alert('⚠️ ' + err.message);
        })
        .returnLoan(row);
    }

    function showMsg(type, text) {
      var m = document.getElementById('msg');
      m.className = 'msg ' + type;
      m.innerText = text;
    }

    function clearForm() {
      var ids = ['item','qty','handler','borrower','dueDate','memo'];
      for (var i = 0; i < ids.length; i++) {
        document.getElementById(ids[i]).value = '';
      }
    }
  </script>
</body>
</html>
```

## 6. 3단계 — 웹앱으로 배포하기

만든 화면을 태블릿에서 열 수 있는 주소(URL)로 만드는 단계입니다.

1. Apps Script 편집기 오른쪽 위 **배포 → 새 배포** 클릭
2. 톱니바퀴 아이콘 → **웹 앱** 선택
3. 설정값:
   - **설명**: `창고 관리 대장 v1` (자유롭게)
   - **다음 사용자로 실행**: `나` (= 본인 Google 계정)
   - **액세스 권한이 있는 사용자**: 아래 6-1 참고
4. **배포** 클릭
5. 처음이면 **권한 승인** 창이 뜹니다. 본인 계정 선택 → "고급" → "(안전하지 않음) 이동" → **허용**
   - 이 앱이 스프레드시트를 편집하고 메일을 보내려면 필요한 정상 절차입니다.
6. 완료되면 나오는 **웹 앱 URL**을 복사해 둡니다. 이 주소가 태블릿에서 열 화면입니다.

### 6-1. 액세스 권한 선택

| 선택지 | 설명 | 추천 상황 |
|---|---|---|
| **나만** | 본인 Google 계정으로 로그인된 기기에서만 접속 | 태블릿을 본인 계정으로 로그인해 쓰는 경우 (보안상 권장) |
| **모든 사용자** | URL만 알면 누구나 접속 가능 | 여러 직원이 로그인 없이 공용 태블릿을 쓰는 경우 |

> 여러 명이 쓰는 공용 태블릿이라면 "모든 사용자"가 편하지만, URL이 외부로 새지 않도록 주의하세요. 가능하면 창고 전용 Google 계정을 하나 만들어 태블릿에 로그인해 두고 "나만"으로 설정하는 방식이 가장 안전합니다.

> 코드를 수정한 뒤에는 **배포 → 배포 관리 → 연필(수정) 아이콘 → 버전: 새 버전 → 배포**를 해야 변경 사항이 반영됩니다. URL은 그대로 유지됩니다.

## 8. 사용 방법

### 입고 / 출고
1. 상단에서 **입고** 또는 **출고** 버튼 선택
2. 물품명, 수량, 담당자 입력 → **기록하기**
3. "✅ OO 완료! 현재고: N개" 메시지 확인

### 대여
1. **대여** 버튼 선택 → 대여자, 반납 예정일 입력칸이 추가로 나타남
2. 물품명·수량·담당자·대여자·반납예정일 입력 → **기록하기**
3. 대여한 수량만큼 현재고에서 빠집니다.

### 반납
1. **반납** 버튼 선택 → 아직 반납되지 않은 대여 목록이 나타남
2. 해당 항목의 **반납 처리** 버튼 클릭
3. 반납된 수량만큼 현재고가 다시 늘어납니다.

모든 거래는 `거래내역` 시트에 한 줄씩 쌓이고, `재고현황` 시트의 현재고가 즉시 다시 계산되며, `iowli@naver.com`으로 알림 메일이 발송됩니다.

### 알림 메일 예시

```
제목: [창고 알림] 입고 - 면장갑 10개

창고 물품 관리 대장 알림
------------------------------
· 구분    : 입고
· 물품명  : 면장갑
· 수량    : 10개
· 담당자  : 홍길동
· 처리시각: 2026-05-21 14:32
· 현재고  : 35개
------------------------------
```

## 10. 자주 묻는 문제 (FAQ)

**"메일이 안 와요"**
스팸함을 먼저 확인하세요. Apps Script 무료 계정의 하루 메일 발송 한도는 100통입니다(창고 운영에는 충분). 코드의 `ADMIN_EMAIL` 주소에 오타가 없는지 확인하세요.

**"시트를 찾을 수 없다는 오류가 나요"**
시트 이름이 코드의 `거래내역`, `재고현황` 과 정확히 일치하는지(앞뒤 공백 포함) 확인하세요.

**"코드를 고쳤는데 화면이 그대로예요"**
배포 → 배포 관리 → 수정(연필) → 버전을 **새 버전**으로 → 배포 해야 반영됩니다.

**"엑셀 파일로 받고 싶어요"**
스프레드시트에서 `파일 → 다운로드 → Microsoft Excel(.xlsx)`을 선택하면 됩니다.

**"재고현황 숫자가 이상해요"**
재고현황 시트는 직접 수정하지 마세요. 거래내역 시트가 원본이며, 거래가 한 번 더 기록되면 자동으로 다시 계산됩니다. 거래내역에서 잘못된 줄을 수정/삭제한 뒤 아무 거래나 한 건 기록하면 재계산됩니다.

## 11. 나중에 추가하면 좋은 기능 (확장 아이디어)

지금 가이드는 "거래 발생 시 알림"까지만 구현합니다. 운영하다 필요해지면 아래 기능을 추가할 수 있습니다.

- **재고 부족 경고**: 현재고가 설정한 최소 수량 이하로 떨어지면 경고 메일 발송
- **반납 기한 알림**: 반납 예정일이 다가온 대여품을 매일 아침 자동 점검해 알림 (Apps Script의 `트리거` 기능 사용)
- **일일/주간 요약 리포트**: 정해진 시간에 하루 거래와 재고 현황을 요약 메일로 발송
- **물품명 자동완성**: 입력 화면에서 기존 물품명을 목록으로 선택
- **바코드/QR 스캔**: 태블릿 카메라로 물품 코드를 읽어 빠르게 입력

이 중 원하시는 기능이 생기면 알려 주세요. 해당 부분의 코드를 추가로 만들어 드릴 수 있습니다.
