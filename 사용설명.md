# 창고 물품 관리 대장 — 사용설명서

---

## 파일 구성

| 파일 | 역할 |
|---|---|
| `offline.html` | 앱 본체 — 브라우저로 열어서 사용 |
| `config.js` | 설정 파일 — URL·파일명 등 수정 시 여기만 |
| `Code.gs` | Apps Script 서버 로직 |
| `Config.gs` | Apps Script 설정 (이메일, 시트명 등) |
| `거래내역.csv` | 모든 거래 기록 (앱이 자동 생성/관리) |
| `재고현황.csv` | 현재 재고 요약 (기록할 때마다 자동 갱신) |

> `config.js`와 `Config.gs`는 민감정보 포함 — git 커밋 제외 (`.gitignore` 등록됨)

---

## 처음 시작하기

1. `offline.html`을 브라우저(Chrome 권장)로 열기
2. **폴더 선택** 버튼 클릭 → CSV 파일을 저장할 폴더 선택
3. 바로 사용 시작

다음부터는 앱을 열면 **이어서 열기** 버튼이 나타납니다. 한 번만 탭하면 자동으로 불러옵니다.

## 기본 사용법

### 모드 선택

| 모드 | 용도 |
|---|---|
| **입고** | 물품이 창고에 들어올 때 |
| **출고** | 물품이 창고에서 나갈 때 |
| **대여** | 물품을 빌려줄 때 |
| **반납** | 빌려준 물품이 돌아올 때 |

> 출고·대여는 현재고보다 많은 수량 입력 시 차단됩니다.

### 기록 순서
1. 모드 선택
2. 물품명 입력 (이전 물품은 자동완성)
3. 수량 입력
4. 담당자 입력
5. **기록하기** 클릭

---

## 초기 재고 설정

**입고** 모드로 초기 수량을 기록합니다.

- 물품명: `면장갑` / 수량: `100` / 메모: `초기재고`

---

## 거래내역 직접 수정

`거래내역.csv`를 엑셀로 열어 직접 수정할 수 있습니다.

- 행 추가·삭제·수정 모두 가능
- **열 순서·열 이름 변경 금지**
- 저장 후 앱 재실행 시 반영

> `재고현황.csv`는 자동 계산 파일 — 직접 수정해도 다음 기록 시 덮어써집니다.

---

## Google Sheets 동기화

| 상황 | 동작 |
|---|---|
| 인터넷 있음 | 기록 즉시 Sheets 전송 + 이메일 알림 |
| 인터넷 없음 | 로컬 CSV만 저장 (동기화 대기) |
| 인터넷 복구 후 다음 기록 | 대기 중이던 것 모두 자동 전송 |

### 전체 재동기화
Sheets 데이터를 오프라인 기준으로 완전히 교체합니다. 이메일 발송 없음.

**전체 재동기화** 버튼 클릭 → 확인

## 설정 변경 (config.js)

```javascript
var SYNC_URL  = '...';             // Apps Script 배포 URL
var APP_TITLE = '창고 물품 관리 대장'; // 앱 이름
var FILE_TX   = '거래내역.csv';     // 거래내역 파일명
var FILE_INV  = '재고현황.csv';     // 재고현황 파일명
var DB_NAME   = 'wh_app';          // 내부 저장소 이름
```

## Apps Script 재배포

`Code.gs` 또는 `Config.gs` 수정 후:

1. [script.google.com](https://script.google.com) → 해당 프로젝트
2. `코드.gs` → Ctrl+A → 삭제 → 최신 `Code.gs` 붙여넣기
3. **배포 → 배포 관리 → 연필 → 새 버전 → 배포**

URL은 바뀌지 않습니다.

---

## 주의사항

- 브라우저는 **Chrome** 권장 (File System Access API 필요)
- `offline.html`과 `config.js`는 **같은 폴더**에 있어야 함
- CSV 파일명은 `config.js`의 `FILE_TX`, `FILE_INV`와 일치해야 함
- `config.js`, `Config.gs`는 **절대 git에 커밋하지 않음** (민감정보)
