# SPEC.md — 창고 물품 관리 대장

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 목적 | 창고 입고·출고·대여·반납을 태블릿에서 버튼 한 번으로 기록, 재고 자동 계산, 알림 메일 자동 발송 |
| 플랫폼 | Google Sheets + Google Apps Script (웹앱) |
| 비용 | 무료 (Google 계정만 필요) |
| 알림 수신 | iowli@naver.com |
| 타임존 | GMT+09:00 서울 |

---

## 2. 아키텍처

```
[태블릿 브라우저 - 웹앱 UI]
        │  google.script.run
        ▼
[Apps Script - Code.gs]
  ├── recordTransaction()  → 거래내역 시트에 기록
  ├── updateInventory()    → 재고현황 시트 재계산
  ├── getOpenLoans()       → 미반납 대여 목록 조회
  ├── returnLoan()         → 반납 처리
  └── sendNotice()         → 알림 메일 발송
        │
        ▼
[Google Sheets]
  ├── 거래내역 시트  (원장 - append only)
  └── 재고현황 시트  (자동 계산 - 매 거래 시 전체 재작성)
        │
        ▼
[MailApp → iowli@naver.com]
```

---

## 3. 데이터 구조

### 3-1. 거래내역 시트 (원장)

| 열 | 항목 | 타입 | 설명 |
|---|---|---|---|
| A | 일시 | Date | 자동 (new Date()) |
| B | 구분 | String | 입고 / 출고 / 대여 |
| C | 물품명 | String | 사용자 입력 |
| D | 수량 | Number | 양의 정수 |
| E | 담당자 | String | 처리한 사람 |
| F | 대여자 | String | 대여 시만 |
| G | 반납예정일 | String/Date | 대여 시만 |
| H | 반납여부 | String | 미반납 / 반납완료 / - |
| I | 반납일 | Date | 반납 처리 시 자동 |
| J | 메모 | String | 선택 입력 |

### 3-2. 재고현황 시트 (자동 계산)

| 열 | 항목 | 계산식 |
|---|---|---|
| A | 물품명 | 거래내역 등장 물품 전체 |
| B | 총입고 | 입고 수량 합계 |
| C | 총출고 | 출고 수량 합계 |
| D | 대여중 | 미반납 대여 수량 합계 |
| E | 현재고 | 총입고 − 총출고 − 대여중 |

> 재고 공식: `현재고 = 총입고 - 총출고 - 대여중(미반납)`

## 4. 파일 구성

```
Google Apps Script 프로젝트
├── Code.gs       — 서버 사이드 로직 (4개 핵심 함수)
└── index.html    — 클라이언트 UI (태블릿 입력 화면)
```

### 4-1. Code.gs 함수 목록

| 함수 | 역할 | 트리거 |
|---|---|---|
| `doGet()` | 웹앱 진입점, index.html 반환 | HTTP GET |
| `recordTransaction(data)` | 거래 기록 → 재고 계산 → 메일 발송 | 클라이언트 호출 |
| `updateInventory()` | 재고현황 시트 전체 재계산 | recordTransaction, returnLoan 내부 |
| `getStock(item)` | 특정 물품 현재고 조회 | 내부 |
| `getOpenLoans()` | 미반납 대여 목록 반환 | 클라이언트 호출 |
| `returnLoan(rowNum)` | 지정 행을 반납완료 처리 | 클라이언트 호출 |
| `sendNotice(...)` | 알림 메일 발송 (실패 시 거래는 유지) | 내부 |

### 4-2. index.html UI 구성

| 컴포넌트 | 설명 |
|---|---|
| 모드 버튼 4개 | 입고(녹색) / 출고(적색) / 대여(청색) / 반납(보라) |
| 입력 폼 | 물품명, 수량, 담당자, (대여 시) 대여자·반납예정일, 메모 |
| 반납 화면 | 미반납 대여 목록 + 반납 처리 버튼 |
| 피드백 메시지 | 성공(녹색) / 오류(적색) 인라인 표시 |

## 5. 핵심 비즈니스 로직

### 5-1. 입력 검증
- 물품명 비어있으면 오류
- 수량 ≤ 0 이면 오류
- 출고/대여 시 요청 수량 > 현재고이면 오류 (재고 부족 방지)

### 5-2. 재고 계산 방식
- `updateInventory()`는 거래내역 전체를 매번 재스캔하여 재고현황을 전체 재작성
- 대여의 재고 차감 조건: `반납여부 === '미반납'` 인 행만 집계

### 5-3. 반납 처리
- 거래내역 시트의 해당 행(rowNum) H열 → `반납완료`, I열 → 현재 시각
- 반납 후 `updateInventory()` 재실행

### 5-4. 알림 메일
- 매 거래(입고·출고·대여·반납) 후 `MailApp.sendEmail()` 호출
- 메일 발송 실패 시 `console.error`만 기록, 거래 기록은 롤백하지 않음
- 무료 계정 일일 한도: 100통

- 재고 최소치 이하 시 경고 메일

- 일/주간 요약 리포트 메일
- 물품명 자동완성 (기존 물품 목록에서 선택)
- 바코드/QR 스캔 입력

## 10. 구현 순서 (Builder 참고)

1. Google Sheets 생성 + 시트 이름 설정 (`거래내역`, `재고현황`)
2. 시간대 설정 (GMT+09:00 서울)
3. Apps Script 편집기에서 `Code.gs` 작성
4. `index.html` 작성
5. 저장 후 웹앱으로 배포 (권한 승인 포함)
6. 태블릿에서 URL 열기 → 홈 화면 추가
7. 테스트 시나리오 8개 순서대로 검증
